Docker basics · Lesson 3

Build and run your first container

Package an app into an image, then start it with a single command.

⏱ ~15 min Needs: Docker installed from Lesson 1 Intermediate Lesson 3 of 9

By the end of this lesson you'll be able to:

  • Build an image from a Dockerfile and tag it
  • Run a container and map a port to your host
  • Inspect running containers and read their logs
Why this matters for your goal: shipping your web app to a server means handing over a reproducible image, not a folder of files — you'll build that image here.

1The idea#

An image is a frozen snapshot of your app and everything it needs to run. A container is a running instance of that image. You write a Dockerfile, then run docker build -t app . to bake it into an image.

Primary source Dockerfile reference — official docs — every instruction and how the build cache works, straight from the docs.

2Run it#

Start the image with docker run -p 8080:80 app to publish its port, then check it's alive with docker ps.

✦ What you earned

You can now turn a project into a portable image and run it anywhere Docker is installed, with the port wired up to your machine.
