Merge vs Rebase — Cheat Sheet

Quick reference for the three-lesson course. Print or bookmark.

Golden rule: never rebase commits that exist outside your local repository.

What each command does to the graph

git merge git rebase
Graph shape Branching (diamond) Linear
Creates merge commit? Yes — two parents No
Existing commit hashes Unchanged New hashes for every replayed commit
Safe on shared branches? Yes No
Rewrites history? No Yes

Decision rules

Situation Command
Branch is local-only; update from main before PR git rebase main
Shared branch needs latest main git merge main
Integrate a finished feature; record the boundary git merge --no-ff feature
Squash WIP commits on a private branch git rebase -i HEAD~N
Push after rebasing git push --force-with-lease

Conflict resolution mid-rebase

1. Edit the conflicted file
Remove <<< / === / >>> markers; keep the correct code
git add <file>
Stage the resolved file — do not git commit
git rebase --continue
Resume — git moves to the next commit; repeat if needed
git rebase --abort
Cancel the rebase entirely; branch returns to pre-rebase state

Recovery after a bad rebase

git reflog
List every HEAD position; find the entry just before the rebase started
git reset --hard HEAD@{N}
Return the branch to that position — complete undo of the rebase
git reset --hard <hash>
Same thing, using the hash directly from the reflog output

The reflog retains entries for 90 days by default. Unreachable commits (the originals from before a rebase) are available until git garbage-collects them.

Command reference

git merge main
Merge main into current branch; creates a merge commit if histories have diverged
git merge --no-ff feature
Merge and always create a merge commit (no fast-forward), even if one isn't needed
git rebase main
Replay current branch's commits on top of main's tip
git rebase -i HEAD~N
Interactive rebase of the last N commits — squash, reword, reorder
git push --force-with-lease
Force-push only if the remote hasn't changed since your last fetch
git reflog
Show every HEAD position ever recorded locally
git log --graph --oneline
Compact ASCII view of the commit graph — see merge vs rebase shapes

The two shapes — at a glance

Merge:
A ── B ── C ── M          ← main (M = merge commit, two parents)
          \   /
           D ─ E

Rebase (then fast-forward merge):
A ── B ── C ── D' ── E'   ← main (linear; D' and E' are new commits)
