Git Merge vs Rebase

Quick reference · Lesson 1 · Lesson 2 · Lesson 3

The one deciding question

Has this branch been pushed AND might a teammate have pulled it?
│
├─ YES → it is SHARED → use git merge to update it
│
└─ NO  → it is PRIVATE → git rebase is safe and cleaner

When in doubt, assume shared and use merge.

The Golden Rule

Never rebase commits that have been pushed to a shared branch. Rebasing rewrites commit hashes; anyone who already has the old hashes now has a diverging history.

What each command does

Property git merge git rebase
History shape Branching graph Linear line
Existing commit hashes Unchanged All change (new parent)
Records when branches joined Yes — merge commit No
Safe on shared branches Yes No
Preferred for PR review cleanliness Sometimes Yes (if private)

Common commands

Task Command
Update private feature branch with latest main git rebase main
Update shared branch with latest main git merge main
Abort rebase mid-way (clean undo) git rebase --abort
Continue after resolving a conflict git add <file> then git rebase --continue
Push after rebasing a private branch git push --force-with-lease
Find pre-rebase state git reflog
Restore to a reflog position git reset --hard HEAD@{n}

Conflict resolution during rebase

  1. Edit the conflicted file — remove the <<<<<<< / ======= / >>>>>>> markers and leave the correct content.
  2. git add <resolved-file> — mark it resolved. Do not run git commit.
  3. git rebase --continue — advance to the next commit.
  4. Repeat steps 1–3 for each subsequent conflict.

To abandon at any step: git rebase --abort — fully restores the branch to its pre-rebase state.

Recovery with git reflog

# Show recent HEAD movements
git reflog

# Find the entry labelled "rebase (start)" — that is before the rebase
# Example output:
# 4e3a1b2 HEAD@{0} rebase (finish): returning to refs/heads/feature
# 91d0fe3 HEAD@{1} rebase (start): checkout main
# 91d0fe3 HEAD@{2} commit: your last commit before rebase  ← this one

# Restore to that point
git reset --hard HEAD@{2}

Reflog entries expire after 90 days by default. Recent mistakes are always recoverable.

Commit graph in 30 seconds

MERGE creates a new commit with two parents:

      A---B---C
     /         \
D---E---F---G---H ← H has parents G and C

REBASE replays commits with a new parent (new hashes):

              A'--B'--C' ← new hashes
             /
D---E---F---G ← main unchanged
