Regex cheat sheet & glossary

The compressed essence of the course, in one printable page. Core, portable (PCRE / JavaScript) flavor — no lookaround, no exotica.

The one mental model

The engine plants a cursor at the start of the text and asks “does my pattern match starting here?” If not, it slides one character right and asks again — left to right, until a match or the end. Everything below is a richer way to answer that question. Most tokens match one character; quantifiers change how many; anchors match a position and consume nothing.

\D \W \Sthe negations of the above\D = [^0-9]

How many — quantifiers

Each attaches to the item immediately before it. Greedy by default; add ? to make it lazy.

TokenRepeats the previous item
*zero or more
+one or more
?zero or one (optional)
{n}exactly n
{n,m}between n and m
{n,}n or more
*? +? ??lazy: as few as possible
Pitfall · greedy “matches too much” <.*> on <a>x</a> grabs the whole string. Greedy .* takes everything, then backtracks to the last >. Want the first tag? Go lazy: <.*?>.

Where — anchors (zero-width)

TokenAsserts position at…
^start of string / line
$end of string / line
\bword boundary (edge of a \w run)

Validate vs. find: \d{5} finds five digits anywhere; ^\d{5}$ asserts the string is exactly five digits. Anchors are the difference between “contains” and “is.”

Structure — groups & alternation

TokenDoes
(…)group and capture (numbered left-to-right by opening paren)
(?:…)group only — no capture slot
a|balternation: match a or b
(ha)+quantifier applies to the whole group → hahaha

Escaping

To match a character that normally has special meaning, put a backslash before it. \. matches a literal dot; \? a literal question mark; \( a literal parenthesis. Inside a class, most metacharacters are already literal — [.?] is a dot or a question mark.

Glossary

Adhere to these terms across all lessons.

TermMeaning
literalA character that matches itself (most characters).
metacharacterA character with special meaning: . * + ? [ ] ( ) { } ^ $ | \.
character classA […] set; matches one character from it.
shorthand classA one-letter class: \d \w \s and negations.
quantifierHow many of the previous item: * + ? {n,m}.
greedyA quantifier that takes as much as possible, then backtracks.
lazyA quantifier (with trailing ?) that takes as little as possible.
backtrackingThe engine giving characters back to let the rest of the pattern match.
anchorA zero-width assertion about position: ^ $ \b. Consumes nothing.
word boundaryThe position between a \w and a non-\w (or string edge): \b.
groupA (…) unit a quantifier or alternation can act on.
captureA remembered sub-match from a (…) group, numbered left-to-right.
non-capturing group(?:…) — groups without creating a capture slot.
alternationThe | “or” between two sub-patterns.
flagA modifier on the whole pattern (e.g. g global, i case-insensitive, m multiline).
