Reference · canonical language

Regex Glossary

The words this course uses, defined once. Lessons stick to these terms.

Regular expressions are a small language for describing text patterns. This glossary fixes the vocabulary so every lesson means the same thing by the same word. Pair it with the cheat sheet.

Core ideas

Regex (regular expression)
A pattern that describes a set of strings. An engine tries to match it against subject text.
Avoid: regexp, expression, "the search thing"

Engine
The code that runs a regex against text. It scans left to right, trying to match the pattern starting at each position.
Avoid: parser, matcher

Match
A specific run of characters in the subject text that the pattern describes. One pattern can produce many matches in one string.
Avoid: hit, result

Subject / sample
The text the engine searches. In these lessons it is the concrete sample string shown beside the pattern.
Avoid: input, the string

Literal
A character that matches itself, with no special meaning — c, 7, a space.
Avoid: plain character

Metacharacter
A character with special meaning in a pattern — . * + ? ( ) [ ] { } ^ $ | \. Escape it with a backslash to match it literally.
Avoid: special char, symbol

Matching one character

Character class
A [...] set that matches one character from the listed options. [^...] is a negated class — one character not in the set.
Avoid: bracket group, set

Shorthand class
A backslash abbreviation for a common class: \d digit, \w word character, \s whitespace. The capital form is its negation.
Avoid: escape code

Repetition

Quantifier
A token that says how many times the thing before it repeats: * + ? {n,m}.
Avoid: repeater, multiplier

Greedy
The default behaviour of a quantifier: match as much as possible, then give characters back only if the rest of the pattern needs them.
Avoid: maximal, hungry

Lazy
A quantifier made minimal by a trailing ? (e.g. .*?): match as little as possible.
Avoid: non-greedy, reluctant

Backtrack
What the engine does when a greedy match went too far: it gives back characters and retries, hunting for a way to satisfy the whole pattern.
Avoid: rewind, retry

Positions and structure

Anchor
A token that matches a position, not a character: ^ start, $ end, \b word boundary. Anchors consume no text.
Avoid: marker, boundary (except \b specifically)

Group
Parentheses (...) that bundle part of a pattern so a quantifier or alternation applies to the whole bundle.
Avoid: bracket, parens-block

Capture / capturing group
A group (...) that also remembers the text it matched, so you can extract it afterward. Numbered left to right by opening paren.
Avoid: saved group

Non-capturing group
(?:...) — groups without remembering. Use when you need the parentheses for structure but don't want to extract.
Avoid: silent group

Alternation
The | operator: match the pattern on its left or its right. The loosest-binding operator; fence it with a group to limit its reach.
Avoid: or, pipe, choice
