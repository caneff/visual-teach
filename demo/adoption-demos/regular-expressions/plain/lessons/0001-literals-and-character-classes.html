Literals & Character Classes

The smallest true mental model: a pattern the engine drags across text, one position at a time.

Why this lesson Your mission is to read and write regexes without guessing. That starts here — with what a pattern is and how a single character (or a small set of them) matches. Quantifiers and structure come in Lessons 2 and 3.

The one idea

A regular expression is a pattern. The engine takes that pattern and tries to match it against your text, scanning left to right: it tries to match starting at position 0, then position 1, and so on. Most of regex is just learning what each piece of the pattern matches at one position.[1]

The simplest pattern is a literal: characters that match themselves. The pattern cat means “a c, then an a, then a t, adjacent.” It matches anywhere that run appears — even buried inside another word.

Match any one: the dot

The metacharacter . matches any single character (except a newline). It still fills exactly one position.[2] So c.t is “c, then anything, then t.”

Match one from a set: character classes

Square brackets define a character class — one character chosen from the set inside. [ch]at matches cat or hat but not bat. A dash gives a range: [a-z], [0-9]. A leading ^ negates it: [^0-9] is “any one character that is not a digit.”[2]

The shorthands you'll see everywhere

Three classes are so common they have abbreviations. Their capital forms are the negations.[2]

ShortMeansSame as
\da digit[0-9]
\wa word character[A-Za-z0-9_]
\swhitespacespace, tab, newline
The key intuition Everything so far matches exactly one position. \d catches a single 4, then a single 2 — not “42” together. Repetition is a separate idea, and it's the whole of Lesson 2.
