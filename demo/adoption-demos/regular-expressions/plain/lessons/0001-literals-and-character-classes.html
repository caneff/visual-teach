Lesson 1 · The smallest true mental model

Literals & character classes

A regex is a pattern an engine drags across your text, one position at a time, asking “does this match starting here?”

The one idea everything is built on

Forget syntax for a second. A regular expression is a tiny matching machine. You hand it a pattern and a string. It plants a cursor at the start of the string and asks a single question:

“Does my pattern match the text starting right here?”

If yes, it reports the match. If no, it shifts the cursor one character to the right and asks again. It keeps sliding rightward until it finds a match or runs off the end. That left-to-right scan is the whole engine at this stage — everything you'll learn is just a richer way to answer that one question.1

Literal characters match themselves

The simplest pattern is just text. The pattern cat means: a c, then an a, then a t, adjacent, in that order. The engine slides along until three characters in a row spell cat.

the category of scattered things

cat finds the first cat — inside “category”. The engine doesn't care about word meaning; it matches characters.

. — the wildcard for one character

A dot matches any single character (except a newline, by default). It still consumes exactly one position.2

cat cot c8t co_t

c.t matches cat, cot, c8t — any one character between c and t. It does not match co_t (two characters in the gap).

[...] — a character class: one from a set

Square brackets define a set; the pattern matches one character that is in that set. [aeiou] matches a single vowel. A range like [a-z] means any one lowercase letter; [0-9] any one digit.1

gray / grey

gr[ae]y matches both gray and grey — the class allows an a or an e in that one spot.

Note · brackets are their own little world Inside [...] most special characters lose their powers. A dot inside a class, [.], just means a literal period. So [.?] matches a literal dot or a literal question mark.

[^...] — negate the set

A caret just after the opening bracket flips the class: match one character that is not in the set. [^0-9] matches any single non-digit.

ab12!

[^0-9] matches each non-digit (a, b, !) but skips the digits.

The shorthands: \d \w \s

These three come up constantly, so they get one-letter names. Their uppercase versions are the negations.1

Pitfall · one box, one character Every tool on this page matches exactly one position. \d matches a single digit, not a whole number; . is one character, not “the rest of the line.” Matching more than one is the job of quantifiers — that's Lesson 2.

Recall first — then check

Don't peek at the table. Answer from memory; the friction is the point — it's what turns “I saw it” into “I know it.”3

Now read a pattern in the wild

You'll see this shape constantly. Decode it before reading on: [A-Za-z0-9_.]. In words: one character that is a letter, a digit, an underscore, or a dot. (Note the dot is literal — it's inside a class.) That's a single “safe” character of a username or filename. Stack one of these per position and you can describe a whole field — once you can say “how many,” which is next.
