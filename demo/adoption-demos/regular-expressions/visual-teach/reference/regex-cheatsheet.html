Regular Expressions · Reference

Regex cheat sheet & glossary

The compressed essence of the three lessons — every token in one place, plus the words for talking about them. Built for the portable JavaScript / PCRE-style flavor. Print it; keep it by the keyboard.

The one model: a regex is a cursor dragged across the text, checking "does the pattern fit starting here?" one position at a time. Every token below just describes what to check at the cursor.

1Character matchers — match one position

TokenMatchesExample

2Quantifiers — how many of the preceding piece

TokenRepeats the piece before it…Example
*zero or morea*
+one or more\d+ → 1742
?zero or one (optional)colou?r
{n}exactly n\d{4} → a year
{n,m}between n and m\d{1,3}
{n,}n or morea{2,}
*? +? ??lazy — as few as possible<.*?>
Greedy by default. A bare quantifier grabs as much as it can, then backtracks. If a pattern "matches too much," your fix is a lazy ?, a negated class like [^>]*, or a boundary anchor.

5Escaping — turn a special into a literal

These characters are special: . [ ] ( ) { } * + ? ^ $ | \. Put a backslash before one to match it literally.

6Recipes — patterns worth memorising

GoalPattern
A whole number\d+
A whole word\w+
An ISO date\d{4}-\d{2}-\d{2}
A quoted string (no overshoot)"[^"]*"
An HTML-ish tag<[^>]+>
A dotted-quad IPv4(\d{1,3}\.){3}\d{1,3}
A whole standalone word\bword\b
Recipes are starting points, not guarantees — e.g. the IPv4 pattern matches 999.999.999.999 too. Tighten when correctness matters; paste into regex101 to see exactly what each piece does.

7Glossary

The shared vocabulary for the whole course. Adhere to these terms.

patternThe regular expression itself — the description the engine tries to match.
matchA stretch of the text that the pattern fits. A pattern can match in many places.
engineThe code that drags the pattern across the text, checking one position at a time.
literalA character that matches itself (most characters).
metacharacterA character with special meaning (. [ ] ( ) * + ? ^ $ | \).
character classA set in […]; matches one character from the set.
quantifierA token (* + ? {n,m}) saying how many times to repeat the preceding piece.
greedyDefault quantifier behaviour: match as much as possible, then backtrack.
lazyQuantifier with a trailing ?: match as little as possible.
backtrackWhen a greedy match gives characters back so the rest of the pattern can fit.
anchorA zero-width token (^ $ \b) matching a position, not a character.
word boundaryThe position between a \w and a non-\w (or string edge).
groupPieces bundled with (…) so a quantifier or | applies to all of them.
captureThe text a (…) group remembers, numbered left to right; used to extract fields.
non-capturing group(?:…) — groups without consuming a capture number.
alternation| — match the left side or the right side.
flagA modifier on the whole pattern: g global, i case-insensitive, m multiline.
