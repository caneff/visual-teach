Regular expressions · Lesson 1

Literals and character classes

A regex is a pattern an engine drags across your text, one position at a time, asking "does the pattern fit starting here?" Today you learn what fits a single character.

⏱ ~12 min Needs: nothing — start here Beginner Lesson 1 of 3

By the end of this lesson you'll be able to:

  • Explain how the engine scans text left to right, position by position.
  • Match exact text with literal characters, and any character with .
  • Match "one character from a set" with [...], [^...], and ranges.
  • Use the shorthands \d, \w, \s and their negations.
Why this matters for your goal: Every regex you've ever pasted is built from these atoms. Once you can read a pattern one character at a time, the copy-paste-and-hope habit starts to dissolve — you'll see what each piece is asking for. See your mission.

1The one idea: the engine scans

Forget syntax for a second. A regex engine takes your pattern and your text, parks itself at the first position in the text, and asks: does the pattern match starting here? If no, it shifts one position to the right and asks again. It keeps sliding until something matches or it runs off the end. That's the whole loop.

So a pattern is a description of "what would have to be true, starting at some position, for this to count as a match." The simplest description is an exact string — a literal.

pattern /cat/
subject the cat sat on the catalog

The literal cat matches the first run of c-a-t. The engine found it at position 4 and stopped — a basic search returns the first match. (It would have found the "cat" in "catalog" too, on a later scan.)

Flavor note — These patterns are written in the common PCRE / JavaScript style, the same core you'll meet in Python, Java, Go, Ruby, and grep. We're staying inside that portable core all course.

2. — any one character

The dot is the first metacharacter: a character with special meaning. . matches any single character (except a newline, by default). It still matches exactly one position — metacharacters describe a character, they don't skip ahead.

pattern /c.t/
subject cat cot cut crrt coast

c.t = c, then any one character, then t. It matches "cat", "cot", "cut" — but not "crrt" (two characters between) and not "coast" (no t right after one character).

The dot is greedy with meaning, not length. A bare . matches a digit, a space, a punctuation mark — anything. That's often too permissive. When you mean "a letter" or "a digit," say so with a character class. Vagueness is how patterns match too much.

3[...] — one character from a set

Square brackets define a character class: a menu of characters, of which the engine must match exactly one. Inside the brackets, list the characters you'll accept. A hyphen makes a range: [a-z] is any lowercase letter, [0-9] any digit, [A-Za-z0-9] any letter or digit.

pattern /gr[ae]y/
subject is it gray or grey?

[ae] matches one character that is either a or e. So this one pattern catches both spellings — still one character at that position.

Put a caret ^ first, inside the brackets to negate the set: [^...] matches one character that is not in the list.

pattern /[^0-9]/
subject 42k99

"One character that is not a digit." The engine slides past 4 and 2 (both digits, rejected) and matches the first non-digit: k.

Inside a class, most metacharacters relax. A . inside [...] is a literal dot; [.?] means "a dot or a question mark," not "any char or a question mark." The characters that still matter inside a class are ^ (only when first), - (range, unless first or last), ], and \.

4Shorthands: \d \w \s

Some classes come up so often they have one-letter shortcuts. Each still matches exactly one character; the capital-letter version is its negation.

Shorthand Matches one… Same as Negation
\d digit [0-9] \D (non-digit)
\w "word" character — letter, digit, or underscore [A-Za-z0-9_] \W (non-word)
\s whitespace — space, tab, newline [ \t\r\n…] \S (non-whitespace)
pattern /\w\w\w-\d\d\d\d/
subject code ABX-4417 issued

Read it left to right: word, word, word, a literal hyphen, then four digits. The hyphen here is outside a class, so it's just a literal dash. Writing the same character four times is clumsy — Lesson 2 fixes that with quantifiers.

Tangible win: you can now read any pattern made of literals and single-character classes — left to right, one position at a time — and say what each atom accepts. That's most of what you see in the wild.

6Recall from memory

Producing the answer yourself — before peeking — is what moves this from "looks familiar" to "I know it." Try each aloud first.

In one sentence, how does the engine decide where a match starts?
It tries the pattern at the first position; if that fails it shifts one character right and tries again, repeating until it matches or runs off the end of the text.
Write a class that matches a single lowercase vowel.
[aeiou] — five characters listed, the engine matches any one of them.
What does \w match that [A-Za-z] does not?
Digits 0-9 and the underscore _. \w is "word character" = letters, digits, and underscore.
Do it for real (5 min): open regex101.com, paste c.t into the pattern box and cat cot cut crrt coast into the test box, and watch the highlights. Then change the dot to [ao] and see the matches shrink. Making it match in front of you is the fastest way to trust the model.

✦ What you earned

A regex is matched one position at a time. Literals match themselves, . matches any one character, [...] / [^...] match one character from (or not from) a set, and \d \w \s are the everyday shorthands. You can read single-character patterns now.
