Regular Expressions · Lesson 1

Literals and character classes

The smallest true mental model: a regex is a pattern the engine drags across your text, one position at a time, asking "does the pattern fit starting here?"

⏱ ~12 min Needs: nothing — start here Beginner Lesson 1 of 3

By the end of this lesson you'll be able to:

  • Explain how the engine scans text left-to-right, position by position.
  • Read literal characters and . in a pattern.
  • Match one character from a set with […] and [^…].
  • Use the shorthands \d, \w, \s and their negations.

1A regex is a cursor crawling your text

Forget syntax for a second. The engine takes your pattern and tries to match it starting at position 0 of the text. If it fails, it slides one character right and tries again. It repeats until something matches or it runs off the end. That sliding cursor is the whole game.

Say the pattern is the literal cat and the text is a cat. The engine tries each start position until the three letters line up:

a…  pattern needs c — got a. Slide right.
a … needs c — got space. Slide right.
a c·a·t — three in a row. Match.
The one idea to keep: a pattern doesn't search "the whole string" in one go. It is checked at a position, character by character. Everything else — classes, quantifiers, anchors — is just describing what to check at the cursor.

2Literal characters match themselves

Most characters in a pattern are literals: the letter a matches the character a, the digit 7 matches 7. A few characters are special (. [ ] ( ) { } * + ? ^ $ | \) — we'll meet them as we go. Everything else is taken at face value.

3. matches any one character

The dot is the first special character worth knowing: it matches any single character (except a newline, by default). It's a wildcard for exactly one position — not zero, not two.

The pattern c.t means "c, then any one character, then t." Watch which words qualify:

Try c.t, then c..t

cat cot c4t c-t cart cut ct
Watch the failures. cart is four letters, so c.t (exactly three) can't span it; ct has nothing between, so the dot has nothing to match. The dot is greedy for exactly one slot.

4A character class […] matches one from a set

A dot is often too loose — you want "any vowel" or "any digit," not "anything." Square brackets define a character class: it matches exactly one character, drawn from the set inside the brackets.

[aeiou]
one character that is a, e, i, o, or u
[0-9]
one character in the range 0 to 9 — a hyphen inside a class means "range"
[A-Za-z]
one letter, upper or lower case — ranges can be stacked
[^aeiou]
one character that is not a vowel — a leading ^ negates the set

5Shorthands: \d \w \s

Three classes come up so often they get shorthand escapes. Each still matches exactly one character:

\d
a digit — identical to [0-9]
\w
a "word" character — a letter, digit, or underscore: [A-Za-z0-9_]
\s
whitespace — a space, tab, or newline
\D \W \S
the negations — the uppercase version means "anything but that"

6Recall — before you peek

Say the answer out loud first, then reveal. Producing it from memory is what makes it stick — recognising it on screen doesn't.

What does . match, and what's the one common exception?
Any single character — exactly one position — except a newline, by default.
How do you say "any character that is not a digit" two different ways?
[^0-9] (negated class) or the shorthand \D.
Inside [a-z], what does the hyphen mean — and how would you match a literal hyphen?
Between two characters it means a range. To match a literal hyphen, put it first or last in the class: [-a-z] or [a-z-].

Stuck? Reveal a walk-through (try first!)
Two uppercase letters then a digit: [A-Z][A-Z]\d. 7ZZ starts with a digit, qq2 is lowercase, A1B has a digit in the middle, and z9 is only a letter-digit — so none of them qualify. You could also write [A-Z][A-Z][0-9]; \d and [0-9] are the same thing.
