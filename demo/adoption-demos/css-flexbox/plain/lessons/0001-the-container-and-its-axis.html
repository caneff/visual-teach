Step 1 — the mental model

The container and its axis

Before you can position anything, you need to know what flexbox even is.

For layout you've been reaching for floats, position: absolute, and margin nudges — tools that were never meant for arranging things in a line. Flexbox is. The whole system starts with one move and two words. Learn them here and the rest of the course is just choices made inside this model.

Today's win Turn a stack of boxes into a deliberate row, flip it to a column, and say out loud which direction is the main axis — every time.

One move: display: flex

Pick any element and set display: flex. Two things happen at once:

  • That element becomes a flex container.
  • Its direct children automatically become flex items, and they line up along an axis instead of stacking.1

That's it. You don't touch the children — declaring the parent a flex container is what lays the children out. Note the word direct: only the immediate children become items. Grandchildren are laid out by their own parent's rules.

.toolbar {
  display: flex;   /* .toolbar is now a flex container */
}
/* every direct child of .toolbar is now a flex item, in a row */

Two words: main axis and cross axis

Items lay out along the main axis. The cross axis is always perpendicular to it. Every flex decision you'll ever make is really the question: “am I working along the main axis or the cross axis?”

Which direction is the main axis? You choose, with flex-direction:

  • flex-direction: row — the default. Main axis runs left → right; cross axis runs top → bottom.
  • flex-direction: column — main axis runs top → bottom; cross axis runs left → right.
The trap to avoid “Main axis” does not mean horizontal. It means whichever way flex-direction points. Switch to column and the main axis becomes vertical. Tying the axes to flex-direction instead of to the screen is the single insight this lesson exists to give you.1
