Lesson 1 · The mental model

The container and its axis

The smallest true model: one container, one main axis, items that lay out along it.

Why this lesson. Your layouts fight back because floats and position have no idea that the things you're arranging belong together. Flexbox starts from the opposite idea: you mark a parent as a container, and its children become a coordinated row (or column) you can steer. Everything else in this course hangs off the model you build here. See the mission →

A flex layout is just three nouns: a container, its items, and the axis they line up along. Get those three and the rest of flexbox is detail.

One line turns a parent into a container

Take any element and give it display: flex. Two things happen at once:

1. The element becomes a flex container.
2. Its direct children — and only the direct children — become flex items, laid out together along a line.

.toolbar {
  display: flex;   /* this element is now a flex container */
}

You don't position the children. You describe how the container should arrange them, and the browser does the placement. That inversion — from "place each box" to "describe the arrangement" — is the whole point. [MDN: basic concepts]

Direct children only A flex item's own children are not flex items — they sit in normal flow until you make their parent a flex container too. Flex is a relationship between one container and its immediate children, nothing deeper.

The two axes

Every flex container has two axes, always perpendicular:

The main axis is the line items lay out along. The cross axis runs across it. Which direction is "main" is your choice — set by flex-direction:

Switch to flex-direction: column and the axes swap: the main axis now runs top→bottom and the cross axis runs left→right. This is the single most useful thing to internalize, because the alignment properties in Lesson 2 are defined relative to these axes — not to "horizontal" and "vertical." Change the direction and the same property points a different way. [web.dev: flexbox]
