Step 3 — the responsive row

Flexible sizing

Why it's called flexbox — items that share space and reflow.

So far your items have kept their natural size. The last piece is the one the system is named for: items can flex — growing to fill leftover space, shrinking to avoid overflow, and wrapping onto new lines when the row runs out of room. This is what turns a static row into a responsive one, and it's the win the whole mission was after.

Today's win Build one real responsive row — a nav bar — that distributes space on purpose and reflows gracefully when the screen narrows. No luck involved.

How items share leftover space

Set the container to flex and lay out three items in a row. There's usually empty space left over on the main axis. The flex properties decide what each item does with it. They live on the items, not the container.1

  • flex-grow — how greedily an item grows into leftover space. 0 = don't grow (default). A unitless ratio: an item with 2 takes twice the share of one with 1.
  • flex-shrink — how readily an item shrinks when there isn't enough room. 1 = may shrink (default); 0 = never shrink.
  • flex-basis — the item's starting size along the main axis before grow/shrink kicks in (e.g. 200px or auto).

You almost never write the three separately. Use the flex shorthand — flex: grow shrink basis — and memorize a few presets:1

.item { flex: 1; }          /* grow to share space equally (= 1 1 0) */
.sidebar { flex: 0 0 200px; } /* fixed 200px, never grow or shrink */
.logo { flex: 0 0 auto; }    /* stay natural size (the default) */
Mental shortcut flex: 1 on several siblings = “divide the row equally between us.” That one declaration replaces a whole pile of width percentages and float clearfixes.

When the row runs out of room: flex-wrap

By default a flex container forces everything onto one line, shrinking items to make them fit (flex-wrap: nowrap). Set flex-wrap: wrap and items that no longer fit drop onto a new line instead — the basis of a row of cards that reflows on a phone.2

Put it together: build a nav bar

This is the payoff — and it deliberately interleaves all three lessons. Follow the checklist; each step names the property and the lesson it came from.

  1. Make the container — display: flex on the <nav>. The links become items in a row. (Lesson 1)
  2. Center vertically — align-items: center so the logo and links share a baseline on the cross axis. (Lesson 2)
  3. Push links to the right — justify-content: space-between pins the logo left and the links right on the main axis. (Lesson 2)
  4. Space the links — gap: 1.5rem between them. (Lesson 2)
  5. Let it reflow — flex-wrap: wrap so links drop to a second line on narrow screens instead of overflowing. (Lesson 3)
.navbar {
  display: flex;              /* 1 — container          */
  align-items: center;        /* 2 — cross-axis center  */
  justify-content: space-between; /* 3 — logo left, links right */
  gap: 1.5rem;               /* 4 — space the items    */
  flex-wrap: wrap;             /* 5 — reflow when narrow */
}
