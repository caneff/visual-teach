Flexible sizing

How items share leftover space, how a row reflows — and building one responsive nav bar on purpose.

Why this lesson. This is the mission in one page: a row that reflows gracefully on a narrow screen, built deliberately rather than by luck. You'll fold in the container/axis model and the distribute/align skill, add two new ideas — flexible sizing and wrapping — and ship a navbar.

So far the boxes kept their natural size. Now we let them flex: grow to fill leftover space, shrink to avoid overflow, and wrap onto new lines when the row runs out of room.

The flex shorthand

Each item carries three settings that decide how it uses main-axis space. You almost always set them through the flex shorthand:

.item {
  flex: 1;              /* grow shrink basis → 1 1 0  */
}
/* expands to: */
.item {
  flex-grow:   1;   /* share of LEFTOVER space to absorb */
  flex-shrink: 1;   /* willingness to give space back     */
  flex-basis:  0;   /* starting size before grow/shrink   */
}
  • flex-grow — when there's spare room on the main axis, how greedily does this item take a share of it? 0 = stay natural size; 1 = take an equal share; 2 = take twice the share of a 1.
  • flex-shrink — when items would overflow, how readily does this one shrink? 0 = never shrink (can cause overflow); 1 = shrink proportionally (the default).
  • flex-basis — the item's size before grow and shrink kick in. auto = use its content/width; a length like 200px = start there.
The three you'll actually type flex: 1 → equal-width columns that fill the row. flex: 0 0 auto → a rigid item that keeps its natural size (great for a logo). flex: 1 1 220px → "aim for 220px, then share leftover and shrink as needed" — the building block of a responsive card. Wrapping: when one line isn't enough

By default a flex row refuses to wrap — items shrink to stay on one line, even past readability. flex-wrap: wrap lets the row break onto multiple lines once items hit their basis. Combine flex-basis with wrap and you get the canonical responsive grid: as many cards per row as fit, the rest flow to the next line. [MDN: wrapping]

.cards {
  display: flex;
  flex-wrap: wrap;
  gap: 1rem;
}
.cards > * {
  flex: 1 1 220px;   /* ≥220px, then grow to fill; wrap when they can't */
}

Applied workflow: build a responsive nav bar

This checklist interleaves all three lessons. Follow it in order; each step is one decision you now know how to make.

  1. Make the container. display: flex on the <nav>. Its links and logo become items on a row main axis. (Lesson 1)
  2. Set the gutter. gap: 1rem for even space between items — no child margins. (Lesson 2)
  3. Align on the cross axis. align-items: center so the logo and links sit on one baseline regardless of height. (Lesson 2)
  4. Push the links to the end. Give the logo flex: 0 0 auto (rigid) and add margin-inline-start: auto to the first link group — or use justify-content: space-between for logo-left / links-right. (Lesson 2)
  5. Let it reflow. flex-wrap: wrap so on a narrow screen the links drop to a second line instead of overflowing. (Lesson 3)
  6. Decide who grows. Want a search box to fill the middle? Give it flex: 1; leave the links at their natural size. (Lesson 3)
nav.site {
  display: flex;
  align-items: center;
  flex-wrap: wrap;
  gap: 1rem;
}
nav.site .logo   { flex: 0 0 auto; }   /* never shrink the brand */
nav.site .search { flex: 1 1 12rem; } /* fill the middle, wrap when tight */
nav.site .links  { display: flex; gap: 1rem; } /* a nested flex row */

Common trap Expecting flex: 1 to respect each item's content width. It doesn't — the 0 basis throws natural width away, so a long label and a short one end up the same size. Want content to count? Use flex: 1 1 auto (basis auto) instead of plain flex: 1.
