Flexbox Cheat Sheet & Glossary

The compressed essence of the course — one page to keep open while you lay things out. Properties grouped by what they live on, plus the vocabulary every lesson uses.

The one rule that explains the rest: a flex container has a main axis (chosen by flex-direction) and a cross axis perpendicular to it. justify-* works along the main axis; align-* works along the cross axis. Learn which axis is which and every property tells you what it does.

1Glossary

These terms are used consistently across every lesson. When a lesson says "main axis," it means exactly this.

Term Meaning
Flex container An element with display: flex. It lays out its direct children as flex items.
Flex item A direct child of a flex container. Grandchildren are not flex items.
Main axis The direction items flow, set by flex-direction: horizontal for row, vertical for column.
Cross axis Always perpendicular to the main axis.
Free space Leftover room on the main axis after items take their base size. justify-content distributes it; flex-grow lets items absorb it.

2Properties on the container

Property What it does Common values
display Makes the element a flex container. flex
flex-direction Chooses the main axis. row (default) · column
justify-content Positions items along the main axis. flex-start · center · flex-end · space-between · space-around · space-evenly
align-items Positions items along the cross axis. stretch (default) · flex-start · center · flex-end
gap Space between items (no edge gaps). 16px · 1rem · any length
flex-wrap Lets items spill onto new lines instead of shrinking forever. nowrap (default) · wrap

3Properties on the items

Property What it does
flex-grow How greedily an item absorbs leftover free space (default 0 = don't grow).
flex-shrink How readily an item gives up space when the row is too tight (default 1 = shrink).
flex-basis The item's starting size before grow/shrink (default auto = its content size).
flex Shorthand: flex: grow shrink basis. Memorise the two you reach for: flex: 1 (= 1 1 0, share equally) and flex: 0 0 auto (stay my natural size).

4Recipes you'll reuse

You want… Put this on the container
Center one box, both directions display:flex; justify-content:center; align-items:center;
Logo left, links right (nav bar) display:flex; justify-content:space-between; align-items:center;
Row of evenly spaced buttons display:flex; gap:1rem; justify-content:space-evenly;
Stack of cards in a column display:flex; flex-direction:column; gap:1rem;
Cards that reflow when narrow display:flex; flex-wrap:wrap; gap:1rem; + flex: 1 1 16rem on items
One item eats the slack (search bar) flex: 1 on that one item
Two traps. (1) justify-content and align-items swap visual direction when you switch to flex-direction: column — they follow the axes, not the screen. (2) Only direct children are flex items; wrapping a child in another <div> removes it from the flow.
