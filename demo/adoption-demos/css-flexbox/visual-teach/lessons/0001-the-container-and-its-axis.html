The container and its axis

The smallest true mental model: one container, one main axis, and items that flow along it.

Step 1 — the model

You can already build a page. What you don't yet have is a reliable way to put things next to each other on purpose. Floats, position, and margin hacks each solve one case and break the next. Flexbox replaces all of them with a single idea, and this lesson is just that idea — no spacing or sizing decisions yet. By the end you'll be able to look at any flex container and say, before you reload, which way its children will flow.

One declaration changes everything

Add display: flex to an element and two things become true at once: that element becomes a flex container, and its direct children become flex items that lay themselves out together. You don't touch the children — turning on the parent is what moves them.

/* before: two divs stack vertically, like every block element */
.toolbar {
  /* nothing — children are block boxes, one per line */
}

/* after: one line does it */
.toolbar {
  display: flex;   /* now .toolbar is a flex container */
}

The dashed box is the container (display: flex). A, B, C are its items. With no other properties, they sit in a row from the start.
The rule that matters

Only direct children become flex items. A grandchild (an element nested inside an item) is laid out by its own parent's rules, not the container's. Flexbox is strictly a parent → direct-children relationship.

The main axis: the direction items flow

Every flex container has a main axis — the line its items are placed along, one after another. By default that axis runs left-to-right, which is why A, B, C landed in a row. You choose the axis with one property, flex-direction:

row → items flow horizontally (default) column → items flow vertically 
.stack {
  display: flex;
  flex-direction: column;   /* main axis now runs top → bottom */
}

Same three items, same container — only flex-direction: column added. The main axis now runs vertically, so the items flow downward.

Nothing about the items changed. You rotated the axis, and the items followed. That is the whole reframe: in flexbox you don't position each child — you set up an axis and let the children flow along it.

The cross axis: always perpendicular

The second axis comes free. The cross axis is simply the one at a right angle to the main axis — you never set it directly, it's defined by whatever the main axis is.

  • Main axis is row (horizontal) → cross axis is vertical.
  • Main axis is column (vertical) → cross axis is horizontal.

You don't need to do anything with the cross axis yet. You just need to hold both lines in your head, because in Lesson 2 each axis gets its own control: one property to position items along the main axis, another to position them across the cross axis. Knowing which axis is which is what makes those two properties obvious instead of a coin toss.

The classic trap

"Horizontal" and "vertical" are not fixed labels in flexbox. The moment you switch to flex-direction: column, the main axis becomes vertical and the cross axis becomes horizontal. Always think main vs. cross, not left-right vs. up-down — otherwise every alignment property will feel like it's behaving backwards.

Retrieval — not recognition — is what makes this stick. Answer before you re-read; a wrong guess here is cheaper than one in real code.

Free recall — close your eyes first

In one sentence each, from memory: (1) what does display: flex turn an element and its children into? (2) what chooses the main axis? (3) how is the cross axis defined?

Reveal the three answers

(1) The element becomes a flex container and its direct children become flex items. (2) flex-direction (row = horizontal, column = vertical). (3) The cross axis is whatever runs perpendicular to the main axis — you never set it directly.

Your tangible win

You can now turn any element into a flex container, flip its main axis between row and column, and name the cross axis without guessing. That's the foundation everything else hangs on. Next you'll use the two axes to finally center a box and space a row of buttons evenly — on purpose.
