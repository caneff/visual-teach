CSS Flexbox · Lesson 1

The container and its axis

The smallest true mental model of flexbox: a container, one main axis, and items that lay out along it.

⏱ ~10 min Needs: HTML + basic CSS Beginner Lesson 1 of 3

By the end of this lesson you'll be able to:

  • Turn an element into a flex container with one declaration.
  • Say which of its elements become flex items — and which don't.
  • Name the main axis and predict how flex-direction rotates it.

1One line makes a container

Pick any element, give it display: flex, and two things happen at once: the element becomes a flex container, and its direct children become flex items that line up in a row.

A container turns its direct children into a row of items

item 2

That's the whole setup. No floats, no clear, no absolute positioning. The children were stacking vertically a moment ago (block boxes do that); the instant the parent is a flex container, they sit side by side.

Only direct children become items. If you wrap a child in another <div>, that wrapper is the flex item now — its contents are one level too deep to feel the container. This is the single most common "why won't it line up?" surprise.

2The main axis is the direction items flow

A flex container has a main axis — the line its items flow along. flex-direction chooses which way that line runs:

  • flex-direction: row (the default) — main axis runs horizontally, items go left-to-right.
  • flex-direction: column — main axis runs vertically, items stack top-to-bottom.

Switching direction doesn't add or remove anything — it just rotates the axis the items ride on. Try it. Flip flex-direction below and watch both the boxes and the axis readout turn:

Notice the boxes never changed — only the axis did. That single rotation is the entire difference between a row and a column in flexbox.

3The cross axis is "the other one"

Perpendicular to the main axis is the cross axis. You don't choose it directly — it's always whatever's left over:

flex-direction: row
main → horizontal
cross ↕ vertical
flex-direction: column
main → vertical
cross → horizontal

Cross axis is always perpendicular — change the main axis and the cross axis follows.

Why two axes matter now: in the next lesson you'll position items along the main axis with one property and along the cross axis with another. If you know which axis is which, you'll never again guess which property to reach for. That's the whole payoff of this lesson.

4Check your model

One recognition check, then a memory check. No peeking for the second.

Which elements become flex items?

Now from memory — answer out loud first, then reveal:

You set flex-direction: column. Which way does the main axis run, and which way is the cross axis?
Main axis runs vertically (items stack top to bottom); the cross axis is horizontal — always perpendicular to the main axis.
Your three buttons stack vertically when you wanted a row. The CSS says display: flex. What's the likely cause?
Either flex-direction is set to column, or the three buttons aren't direct children of the flex container — they're wrapped in another element, so that wrapper is the only flex item.

✦ What you earned

display: flex turns an element into a container whose direct children become items flowing along a main axis; flex-direction rotates that axis between row and column, and the cross axis is always perpendicular. You can now make a row on purpose.
