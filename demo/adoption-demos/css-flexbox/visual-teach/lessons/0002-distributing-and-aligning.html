CSS Flexbox · Lesson 2 of 3

Distributing & aligning

Two properties — one per axis — and the centering problem is finally over.

Step 2 — the workhorse

In Lesson 1 you set up a container with a main axis and a perpendicular cross axis. Today each axis gets one control. Master these two and you can center anything and space a row evenly — the skills the whole mission was built around.

The whole lesson in two lines

justify-content positions items along the main axis. align-items positions them along the cross axis. That's the pair. Everything below is just learning their values.

justify-content — spacing along the main axis

This is how you distribute items in the direction they flow. With a default row, that means horizontally. Six values cover essentially every layout you'll reach for:

.bar {
  display: flex;
  justify-content: space-between;
}

align-items — alignment along the cross axis

Same idea, other axis. With a default row the cross axis is vertical, so align-items controls how items sit top-to-bottom inside the container's height.

center — items centred on the cross axis. The other half of centering a box.

The values mirror justify-content's ends: flex-start pins items to the cross-axis start (top, in a row), flex-end to the end (bottom).

Finally: centering a box, both directions

Center on the main axis and the cross axis at once, and the item sits dead-centre — no margins, no position, no math:

.center-box {
  display: flex;
  justify-content: center;  /* main axis  → horizontal */
  align-items: center;      /* cross axis → vertical   */
}
★
One item, both properties set to center. The decades-old "how do I vertically center a div" problem, solved in two lines.

gap — space between, the clean way

To put breathing room between items, don't reach for margins on each child. Set one property on the container:

.row {
  display: flex;
  gap: 1rem;   /* space between items only — never outside edges */
}
1
2
3
gap: 1rem — uniform space between items, none added at the container's outer edges. No more margin-on-every-child-except-the-last hacks.
Don't confuse them

gap sets a fixed distance between items. justify-content: space-* distributes leftover space, so its gaps grow and shrink with the container. They compose: a common pattern is gap for a guaranteed minimum plus space-between to push groups apart.

The decision loop — read the layout, pick the property

This is the skill itself: not reciting values, but mapping a target layout to the property that produces it. Decide each one before you tap.

Decision · 1 of 4

A row of nav links: logo hard against the left edge, the account menu hard against the right, nothing in the middle. Which property?

Decision · 2 of 4

Decision · 3 of 4

Five buttons in a wide bar, and you want identical spacing everywhere — between each pair and at both outer edges. Which property?

Free recall — from memory, no scrolling

Name the one property for each job: (1) push items apart along the row; (2) align items up/down in a row; (3) put a fixed space between items.

Reveal

(1) justify-content — main axis. (2) align-items — cross axis. (3) gap. If you can recite "justify = main, align = cross, gap = between," you have the whole lesson.

Your tangible win

You can now space a row any way a design asks for, center a box in two honest lines, and choose the right property from the look you want instead of guessing. Next, items stop being fixed-size: you'll let them share leftover space and wrap onto new lines, turning today's row into a responsive one.
