Flexible sizing

Items share leftover space and wrap when they run out — the "flex" in flexbox, and the last piece of a responsive row.

Step 3 — applied

So far your items have kept their natural size. The reason flexbox is called flex is that items can flex — give back space when the container is tight, take more when it's roomy. Add the ability to wrap onto new lines and you have a row that reflows gracefully, which is exactly the mission. This lesson interleaves Lesson 1 and Lesson 2 into one real component.

The flex shorthand: three numbers, one property

Sizing lives on the items, not the container, via the flex shorthand. It bundles three longhands:

  • flex-grow — how greedily an item takes a share of leftover space. 0 = never grow; 1 = take an equal share; 2 = take twice the share of a 1.
  • flex-shrink — how readily it gives space back when the container is too small. 1 (default) = shrinks; 0 = refuses to shrink below its basis.
  • flex-basis — the item's starting size along the main axis before growing or shrinking. auto = use its content/width; or a length like 200px.
.item {
  flex: 1 1 0;     /* grow:1  shrink:1  basis:0  → equal-width columns */
}

/* the three names you'll actually reach for: */
flex: 1;        /* = 1 1 0   → share all space equally */
flex: auto;     /* = 1 1 auto → grow from natural size */
flex: none;     /* = 0 0 auto → fixed; never grow or shrink */

The mental model

flex-grow divides up the extra space, not the total width. With three items at flex: 1, the leftover after their bases is cut into three equal slices. Change one to 2 and it claims two of (now) four slices. You're setting ratios, not pixels — which is what makes the layout adapt on its own.

flex-wrap: let the row break onto new lines

By default a flex container forces every item onto one line, shrinking them until they fit (or overflow). That's flex-wrap: nowrap. Flip it to wrap and items that don't fit drop to a new line instead — the heart of a responsive row.

.cards {
  display: flex;
  flex-wrap: wrap;        /* overflow items move to the next line */
  gap: 1rem;
}
.cards > * {
  flex: 1 1 200px;    /* grow & shrink, but aim for ~200px wide */
}

Why your row overflowed instead of wrapping

The single most common surprise: nothing wraps until you set flex-wrap: wrap. The default nowrap keeps everything on one line and lets items overflow once they can't shrink further. If a row is spilling off-screen, this property is almost always the fix.

Applied: build a responsive nav bar

Time to put all three lessons together. Work through this checklist against a real file — it ticks persist if you close the tab and come back. The goal: a nav bar with a logo on the left, links that space out, and that wraps cleanly on a narrow screen.

Here's a reference implementation to check yours against:

nav {
  display: flex;
  align-items: center;
  flex-wrap: wrap;
  gap: 1rem 1.5rem;        /* row-gap  column-gap */
}
nav .logo   { margin-right: auto; }  /* push everything else right */
nav .search { flex: 1 1 12rem; }     /* grows; wraps below ~12rem */
nav .btn    { flex: none; }          /* buttons keep their size */

Free recall — the whole course in four answers

From memory: (1) what makes children flex items? (2) which property positions along the main axis? (3) which along the cross? (4) which two properties make a row responsive?

Reveal

(1) display: flex on the parent. (2) justify-content. (3) align-items. (4) flex (so items share space) + flex-wrap: wrap (so they reflow onto new lines). That's the entire model — container, two axes, distribute, align, size, wrap.
