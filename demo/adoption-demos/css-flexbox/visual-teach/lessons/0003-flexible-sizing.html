CSS Flexbox · Lesson 3

Flexible sizing

The applied payoff: how items share leftover space, how a row reflows onto new lines, and how to assemble one responsive row on purpose.

⏱ ~18 min Needs: Lessons 1–2 Intermediate Lesson 3 of 3

By the end of this lesson you'll be able to:

  • Read the flex shorthand and know what flex: 1 actually does.
  • Make one item absorb the leftover space (a stretchy search bar).
  • Let a row reflow onto multiple lines with flex-wrap.
  • Follow a short checklist to build a responsive nav bar end to end.
Why this matters for your goal: "responsive rows that reflow gracefully feel like luck" — this is where the luck ends. Lessons 1 and 2 placed and aligned items at a fixed size. Now the items themselves flex: they share extra room, give up room when it's tight, and wrap when the row runs out. That's the difference between a layout that survives a phone screen and one that overflows it.

1Free space, and who absorbs it

After items take their natural size, a row usually has leftover room on the main axis — free space. In Lesson 2, justify-content distributed that space around the items. Now we let the items themselves soak it up, with three properties — almost always written as one shorthand, flex:

the flex shorthand
.item {
  flex: 1;          /* shorthand for: flex-grow:1  flex-shrink:1  flex-basis:0 */
}

/* the three parts, spelled out */
.item {
  flex-grow: 1;     /* how greedily I absorb leftover space (0 = don't grow) */
  flex-shrink: 1;   /* how readily I give up space when it's tight (default 1) */
  flex-basis: 0;    /* my starting size before grow/shrink kicks in */
}

Two values cover almost everything you'll write: flex: 1 ("share the space equally with the other flex: 1 items") and flex: 0 0 auto ("don't grow, don't shrink — stay my natural size").

flex-grow is a ratio, not a width. If three items are flex: 1, flex: 2, flex: 1, the leftover space splits 1 : 2 : 1 — the middle item takes twice the extra, not twice the total. Give one item flex: 1 and leave the rest at the default 0, and that one item eats all the slack — exactly how a search box stretches to fill a toolbar.

Set just the middle box to flex-grow: 1 and leave the others at 0: the search field stretches, the buttons stay put. That's the single most useful sizing move in flexbox.

2flex-wrap — let the row breathe onto new lines

By default a flex row never wraps: when space runs out, items shrink (and keep shrinking past comfort). flex-wrap: wrap lets them spill onto a second line instead — the foundation of a card grid that survives a narrow screen.

Toggle flex-wrap below, then narrow your browser window (or zoom in). With nowrap the items cram and shrink; with wrap they reflow onto new rows:

The responsive-card idiom. Pair wrap with a flex basis: flex-wrap: wrap on the container and flex: 1 1 16rem on each card means "be about 16rem wide, grow to fill a row, and wrap to the next line when fewer than that fit." One rule, and the cards reflow from four-up to one-up as the screen narrows — no media queries.

3Put it together — a responsive nav bar

This interleaves all three lessons: a container with an axis (L1), distribution and alignment (L2), and flexible item sizing (L3). Work through the checklist — each step is one declaration, building to the finished bar. Progress is saved if you step away.

  • Make the bar a flex container On the <nav>, set display: flex. Its direct children — logo, links, sign-in button — become flex items in a row. (Lesson 1.)
  • Center everything vertically Add align-items: center so the logo, text links, and button share one baseline regardless of their heights. (Lesson 2 — cross axis.)
  • Add even spacing Put gap: 1rem on the container so items don't touch — no per-child margins to clean up. (Lesson 2.)
  • Push the sign-in button to the far right Give the spacer/links group flex: 1 — that one item absorbs all the free space, shoving the button to the edge. (Or use justify-content: space-between if the bar has exactly two groups.) (Lesson 3.)
  • Let it reflow on small screens Add flex-wrap: wrap so the bar drops the button to a second line instead of overflowing when the viewport is tight. (Lesson 3.)

Here's the rule that checklist builds — the whole bar in six declarations:

nav-bar.css
nav {
  display: flex;          /* L1 — container; children become items   */
  align-items: center;    /* L2 — center on the cross axis            */
  gap: 1rem;              /* L2 — even spacing, no edge gaps          */
  flex-wrap: wrap;        /* L3 — reflow instead of overflow          */
}
nav .links {
  flex: 1;                /* L3 — eat the slack, push sign-in to edge */
}

4Lock it in

In a toolbar, you want one search box to absorb all the leftover space while the buttons keep their size. What do you put on the search box?

From memory first, then reveal:

What three properties does flex: 1 expand to, and what's the headline effect?
flex-grow: 1, flex-shrink: 1, flex-basis: 0. Effect: the item grows from a zero base to take an equal share of the row's free space (equal-width columns when every item has it).
Your row of cards overflows the screen on a phone instead of stacking. One property fixes it — which?
flex-wrap: wrap on the container. Pair it with flex: 1 1 16rem (or similar) on the cards so they reflow to fewer-per-row as the screen narrows.
Test it for real. Open a page you've built, find a row you once forced with floats or position, and rebuild it with these three lessons. Then bring the gnarliest case to a community — r/css or the flexbox tag on Stack Overflow — and have practitioners critique it. Real layouts under real eyes is where the model hardens into wisdom.
