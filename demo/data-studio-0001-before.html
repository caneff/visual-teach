Looker Studio · Lesson 1

Reusable Data Sources

One data source, many reports — fix a field once, everywhere updates.

⏱ ~20 min · Needs: a Looker Studio account, a Google Sheet or BigQuery table

1What is a data source?

A data source is Looker Studio's metadata layer over your raw data. It stores field names, types, and aggregation rules — not the data itself. Every report chart draws from a data source, not directly from a spreadsheet or database.

Key idea: A data source is a saved connection with a field schema on top. Change the schema once and every report that references it updates automatically.

2Creating a reusable data source

A data source created inside a report is embedded — it belongs only to that report and cannot be reused. To share it across reports, create it from the main Data Sources page.

Common mistake: creating the data source from inside a report. It looks the same, but it is embedded and invisible to other reports.
  1. Open Data Sources Go to datastudio.google.com → top-left menu → Data Sources.
  2. Create → pick a connector Click +, choose Google Sheets (or BigQuery), and authorise.
  3. Fix field types Change Date fields from Text → Date; set aggregation on numeric fields.
  4. Click Done The data source is now available to any report in the same Google account.

3Connecting a report to the shared source

With your reusable data source saved, every new report can reference it instead of re-creating a connection.

Payoff: if you later add a calculated field or fix a type, every report using this source picks up the change automatically.

In the report editor: Resource → Manage added data sources → Add a data source, then pick the one you created. Avoid Create new data source from this dialog — that creates an embedded source.
